# 디자인 에셋 생성 가이드

앱인토스 콘솔 등록에 필요한 이미지 에셋을 자동으로 만들어 주는 기능이다.

## 무엇을 왜 만드는가

콘솔에 앱을 등록하려면 아래 이미지가 반드시 필요하다. 사양을 만족하지 못하면 심사에서 반려된다.

| 에셋 | 크기 | 왜 필요한가 |
|---|---|---|
| 앱 로고 | 600×600 | 토스 홈·검색 리스트 등 작은 영역에 노출. 48×48까지 축소돼도 또렷해야 함 |
| 가로형 썸네일 | 1932×828 | 배너형 노출 영역용 |
| 세로 스크린샷 3장 | 636×1048 | 콘솔 등록 시 우선 노출 자료. 실제 구현된 UI 증거물 — `/ait-screenshots` 가 implement 후 자동 생성 |
| 가로 스크린샷 (선택) | 1504×741 | 가로 모드 노출용 보조 자료, 1장 이상 |
| 정방형 썸네일 (선택) | 1000×1000 | 콘솔 노출에 사용되지 않음 — 자동 생성 대상에서 제외 |

추가로 **앱 내부에서 쓰는 캐릭터·일러스트**는 PRD에 명시된 경우에만 따로 생성된다.

## 어떻게 쓰는가

### 방법 1: 슬래시 커맨드

앱 폴더(`apps/<앱-이름>/`)에서 Claude Code로 실행한다.

```
/ait-assets
```

인자 없이 쓰면 현재 준비 상태를 점검하고 없는 것만 만든다. 옵션이 두 개 있다.

```
/ait-assets game                  # 게임 앱 (체크리스트가 달라짐)
/ait-assets partner               # 비게임 앱
/ait-assets --generate-images     # 이미 있어도 다시 만들지 물어봄
```

### 방법 2: 대시보드 버튼

`pnpm dev` 로 대시보드를 띄우고 앱 상세 페이지로 간다. 로고·썸네일이 비어 있으면 `/ait-assets` 버튼이 보이고, 클릭하면 자동 실행된다.

### 방법 3: 전체 출시 플로우의 일부

`/ait-launch` 로 기획부터 빌드까지 돌리면 2단계에서 자동으로 실행된다. 따로 챙길 필요 없음.

## 생성 방식별 특징

에셋마다 생성 방식이 다르다 — 각자 최적인 도구를 쓴다.

- **로고는 벡터 SVG**로 만든다. 작게 줄여도 또렷해야 하므로 회화적 이미지(DALL-E 류)는 쓰지 않는다. 단순한 기하 도형으로 서비스를 상징한다.
- **썸네일은 HTML/CSS로 앱 UI를 재현**해서 헤드리스 브라우저로 캡처한다. 앱의 핵심 플레이 화면을 전달하기 위해서다.
- **스크린샷은 실제 dev 서버를 띄워서 캡처**한다. 실제 구현 증거여야 하므로 목업이 아니라 진짜 화면이어야 한다.
- **캐릭터·일러스트는 OpenAI 이미지 API**를 쓴다. 기하 도형이나 UI 목업으로 표현할 수 없는 회화적 묘사가 필요할 때만.

## 결과물

실행이 끝나면 앱 폴더 아래에 파일이 생긴다.

```
apps/<앱-이름>/
├── assets/                  # 콘솔 등록용
│   ├── logo.png             # 600×600
│   ├── thumbnail-wide.png   # 1932×828
│   └── screenshots/         # 선택
└── src/
    └── assets/              # 앱 내부용 (캐릭터·일러스트)
```

`.meta-dashboard.json` 의 경로 필드도 자동으로 채워진다.

## 미리 준비해둘 것

- **PRD 또는 앱 아이디어 설명** — 어떤 서비스인지 알아야 적절한 로고·썸네일이 나온다
- **`granite.config.ts` 의 `brand.primaryColor`** — 브랜드 컬러. 없으면 토스 기본 블루 사용
- **스크린샷까지 원하면 dev 서버 기동** — `pnpm --filter @barreleye/<앱-이름> dev`
- **캐릭터·일러스트까지 원하면 `.env` 에 `OPENAI_API_KEY`** — `.env.example` 복사 후 키 입력

환경이 제대로 잡혀 있는지 한 번에 확인하고 싶으면 `pnpm preflight` 를 실행한다.

## 결과가 마음에 안 들면

생성 중 대화로 방향을 바꿀 수 있다. 예: "더 미니멀하게", "일러스트 느낌 빼고", "로고는 그대로 두고 썸네일만 다시".

로고가 너무 범용적(돋보기·톱니 같은)으로 나오면 서비스 맥락을 한두 줄 더 알려주면 개선된다.

## 텍스트 리소스 주의

이미지 외에 콘솔에 입력할 **앱 이름·부제·상세 설명**도 스킬이 함께 점검한다. 공통 규칙은 다음과 같다.

- 허용 특수문자: `:`, `∙`, `?` 만
- 이모지 사용 불가
- 상세 설명은 선택이지만 **AI 마케팅 소재 생성에 쓰이므로 구체적으로** 쓸수록 좋다

자세한 콘솔 등록 규격·텍스트 필드 규칙은 [리소스 체크리스트](../launch-flow/02-resource-checklist.md) 참고.
