# 1단계: 기획 (Planning Guide)

## 목적

앱 아이디어를 구체화하고, 앱인토스 정책에 부합하는지 사전 검토한다.

## 기획서 필수 항목

기획서에 아래 항목이 포함되어야 한다:

1. **앱 개요**: 한 줄 소개, 핵심 가치
2. **타겟 사용자**: 주요 사용층 정의
3. **핵심 기능**: MVP 기능 목록 (우선순위 포함)
4. **수익화 모델**: 인앱 결제 / 인앱 광고 / 토스페이 / 프로모션 / 없음
5. **앱 유형**: 게임 / 비게임
6. **기술 구조**: WebView / React Native / Unity

## 서비스 오픈 정책 - 제한 서비스

아래 서비스는 **등록 불가**:

- 디지털 자산 및 가상자산 거래
- 자금세탁 가능성 서비스
- 불법 행위 조장 서비스
- 사행성 및 복권/베팅성 콘텐츠
- 금융 상품 중개/판매/광고
- 투자 자문 및 유료 정보 제공
- 기존 회사/서비스를 단순 홍보하기 위한 앱
- 이외 내부 정책상 승인 불가 서비스 (심사 결과는 내부 정책에 따라 변경될 수 있음)

## 자격이 필요한 카테고리

- **의료**: 비대면 진료 수익 서비스 불가
- **쇼핑몰**: 고객센터/가품 방지/환불 정책 필수
- **교육**: 국가 자격증 없으면 수익 불가

## 서비스별 특수 정책

| 서비스 유형   | 핵심 정책                                                                     |
| ------------- | ----------------------------------------------------------------------------- |
| 게임 (일반)   | 게임물관리위원회 등급분류 필수, 사행성 금지                                   |
| 게임 (웹보드) | 위 + 만 19세 이상 성인 인증 필수, 결제 한도 관리, 확률형 아이템 확률 공개     |
| 만남/소개팅   | 만 19세 이상만, 성인 인증, 신고/모니터링 체계                                 |
| AI 채팅/상담  | 위험 키워드 필터, 안전 응답 프로토콜, AI 모델/프롬프트 변경 시 사전 배포 검토 |
| 채팅 서비스   | 신고/차단 기능, 불법 행위 방지, 메시지 암호화                                 |
| 정치 서비스   | 원문 그대로 중립 전달, 유저 참여 기능 일체 금지                               |
| 민감 콘텐츠   | 선정성/폭력성 제한, 진입 전 안내 화면 필요                                    |

## 생성형 AI 서비스 정책

생성형 AI를 활용하는 서비스는 반드시 아래를 준수해야 함:

- **사전 고지 의무**: AI 활용 사실을 최초 이용 시점에 고지 필수
- **표시 의무**: AI 결과물에 라벨/배지/워터마크 등 즉시 인식 가능한 표시 필수
- 위반 시 최대 **3,000만 원 과태료**, 현장 조사, 서비스 중지/시정명령 가능

## 자사 앱 설치 / 외부 링크 정책

- 자사 앱 설치 유도 **절대 금지** (배너, 링크, 혜택 안내 등 모든 형태)
- 외부 링크 이동 **제한적 허용** (법률 고지/공공기관 페이지 등만 허용)
- 미니앱 내 기능 완결성 요구: 콘솔에 설정한 "앱 내 기능"은 미니앱 내에서 완결 가능해야 함

## 어뷰징 방지 정책

- 동일 워크스페이스에서 **유사한 핵심 기능 기반의 앱을 여러 개 출시할 수 없음**
- **동일 브랜드 중복 출시 제한**: 이름/로고/색상 등 브랜드 아이덴티티를 반복하여 여러 미니앱 출시 불가
- 예외: 서비스 목적과 핵심 기능이 명확히 다르면 동일 브랜드라도 별도 출시 가능
- 위반 시: 콘텐츠 수정 요청 → 서비스 미노출 → 반복 시 등록/운영 제한

## 로그인/결제/광고 정책

- **로그인**: 토스 로그인만 사용 가능
- **결제**: 토스페이(실물), 인앱결제(디지털)만 허용
- **광고**: 앱인토스 전면형/보상형/배너 광고만 사용

## 기획 검토 체크리스트

- [ ] 제한 서비스에 해당하지 않는가?
- [ ] 자격이 필요한 카테고리라면 요건을 충족하는가?
- [ ] 서비스 유형별 특수 정책을 확인했는가?
- [ ] 수익화 모델이 정책에 부합하는가?
- [ ] MVP 기능 범위가 명확한가?
- [ ] 앱 유형(게임/비게임)이 결정되었는가?
- [ ] 개발 방식(WebView/RN/Unity)이 결정되었는가?
