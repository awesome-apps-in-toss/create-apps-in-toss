# 앱인토스 미니앱 출시 체크리스트

> 대시보드 없이 개인이 직접 체크할 수 있는 단계별 체크리스트.
> 각 단계에서 사용할 수 있는 Claude 스킬(`/ait-*`)을 함께 표기했다.

---

## STEP 1. 구상하기

> 아이디어를 구체화하고, 앱인토스에서 만들 수 있는지 확인하는 단계

| 스킬 | 설명 |
|------|------|
| `/ait-plan` | 아이디어를 대화형으로 검증하고, 정책 검토 후 PRD 문서를 자동 생성 |
| `/ait-preflight` | 개발 환경(MCP, 도구, env)이 준비됐는지 사전 점검 |

- [ ] 앱 한 줄 소개 작성 (이 앱이 뭔지 한 문장으로)
- [ ] 타겟 사용자 정의 (누가 쓸 앱인지)
- [ ] 핵심 기능 목록 정리 (MVP 기준, 우선순위)
- [ ] 앱 유형 결정: 게임 / 비게임
- [ ] 수익화 모델 결정: 인앱 결제 / 인앱 광고 / 토스페이 / 없음
- [ ] **정책 검토**: 제한 서비스에 해당하지 않는지 확인
  - 가상자산 거래, 금융 상품 중개, 사행성 콘텐츠, 단순 홍보 앱 등은 등록 불가
- [ ] (게임) 게임물관리위원회 등급분류 준비
- [ ] (만남/AI채팅/정치 등) 해당 카테고리 특수 정책 확인
- [ ] PRD(기획 문서) 작성 완료

---

## STEP 2. 이름·브랜딩 확정하기

> 한번 정하면 못 바꾸는 것들이 있으니 신중하게

| 스킬 | 설명 |
|------|------|
| `/ait-plan` | STEP 1에서 이어서 진행하면 브랜딩 항목도 PRD에 포함됨 |

- [ ] **appName 확정** (⚠️ 등록 후 변경 불가! 영문 소문자+하이픈)
- [ ] 한국어 앱 이름 확정 (서비스 성격이 바로 이해되는 이름)
- [ ] 영어 앱 이름 확정 (15자 이내, 명사형, 동사구 불가)
- [ ] 브랜드 컬러 HEX 코드 확정 (예: `#3182F6`)
- [ ] 카테고리 선택 (대분류 > 중분류 > 소분류)
- [ ] 부제 작성 (사용자가 얻는 가치를 구체적으로)

---

## STEP 3. 개발하기

> 코드를 작성하고 동작하는 앱을 만드는 단계

| 스킬 | 설명 |
|------|------|
| `/ait-scaffold` | 프로젝트 기본 틀 생성 (React + Vite + granite), 라우팅/쿼리/TDS 자동 판단 설치 |
| `/ait-add-routing` | React Router 설치 및 기본 페이지 구조 세팅 |
| `/ait-add-query` | TanStack Query 설치 및 서버 데이터 연결 |
| `/ait-tds-setup` | 토스 스타일 UI(TDS) 컴포넌트 설치 및 Provider 래핑 |
| `/ait-implement` | PRD 기반 기능 구현 (병렬 에이전트 지원) |

- [ ] 프로젝트 스캐폴딩 완료 (`granite.config.ts` 생성됨)
- [ ] `granite.config.ts`에 displayName, primaryColor, icon 설정
- [ ] (비게임) TDS 컴포넌트 설치 및 적용 (`@toss/tds-mobile` 필수)
- [ ] (비게임) `TDSMobileAITProvider`로 앱 래핑
- [ ] 핵심 기능 구현 완료
- [ ] (비게임) 내비게이션 바 정상 동작 (뒤로가기, 브랜드 로고 표시)
- [ ] (비게임) 앱 내 기능 최소 1개 등록
- [ ] (게임) 사운드 On/Off, 풀스크린, 닫기 버튼 등 구현
- [ ] Safe Area 침범 없음
- [ ] 모든 화면에서 나갈 수 있는 방법 존재
- [ ] 외부 링크 없음 (법률 고지/공공기관 제외)
- [ ] iframe 미사용 (YouTube 제외)
- [ ] 타입체크 통과 (`pnpm typecheck`)
- [ ] 린트 통과 (`pnpm lint`)

---

## STEP 4. 출시 에셋 준비하기

> 콘솔에 등록할 이미지와 텍스트를 만드는 단계

| 스킬 | 설명 |
|------|------|
| `/ait-assets` | 로고, 썸네일, 스크린샷 등 이미지/텍스트 리소스 자동 생성 및 점검 |
| `/ait-meta` | `.meta-dashboard.json` 자동 생성 (대시보드 연동용 메타데이터) |

**이미지**

| 항목 | 규격 | 필수 |
|------|------|------|
| 앱 로고 | 600×600 px, PNG, 배경색 필수(투명 불가) | O |
| 정방형 썸네일 | 1000×1000 px, PNG | O |
| 가로형 썸네일 | 1932×828 px, PNG | O |
| 스크린샷(세로) | 636×1048 px, PNG, 최소 3장 | 선택 |
| 스크린샷(가로) | 1504×741 px, PNG, 최소 1장 | 선택 |

- [ ] 앱 로고 준비 (다크/라이트 모드 모두 대응, 토스 리소스 사용 불가)
- [ ] 정방형 썸네일 준비 (핵심 기능 화면)
- [ ] 가로형 썸네일 준비
- [ ] 스크린샷 준비 (선택이지만 제출 시 우선 노출됨)

**텍스트** (특수문자는 `:` `∙` `?` 만 가능, 이모지 불가)

- [ ] 상세 설명 작성 (사용자 경험 흐름: 접속→행동→결과)
- [ ] 검색 키워드 정의 (사용자가 떠올릴 만한 단어)
- [ ] 고객센터 이메일 준비
- [ ] 고객센터 전화번호 준비
- [ ] 고객센터 채팅 상담 주소 준비 (채널톡 등)

**게임 전용 추가**

- [ ] 게임물 등급분류 증빙 (PDF 또는 스토어 URL)
- [ ] 대표자 인감/사인 이미지
- [ ] 게임 플레이 스크린샷 4장

---

## STEP 5. 검수하기

> 출시 기준에 맞는지 셀프 체크하는 단계

| 스킬 | 설명 |
|------|------|
| `/ait-review` | 검수 체크리스트 기반 코드 리뷰 (light: 핵심만 / full: 전수검사) |
| `/ait-ut` | 앱인토스 사용자 페르소나로 사용성 테스트(UT) 시뮬레이션 |

- [ ] (light) 필수 설정 확인: granite.config.ts, TDS, 권한
- [ ] (light) 코드 품질: 타입체크, 린트, 보안 취약점
- [ ] (light) 핵심 UX: Safe Area, 외부 링크, iframe
- [ ] (full) 앱이 정상적으로 열리는지 (게임: 10초 이내)
- [ ] (full) 인터랙션 반응 2초 이내
- [ ] (full) 종료 후 재진입 시 데이터 유지
- [ ] (full) 비속어, 은어, 과도한 유행어 없음
- [ ] (full) 진입 시 바텀시트 자동 오픈 없음
- [ ] (full) CTA 버튼 의도가 명확
- [ ] (full, 비게임) 내비게이션 바 규격 준수
- [ ] (full, 게임) 사운드/햅틱/풀스크린/닫기 버튼 동작
- [ ] (해당 시) 결제/광고/로그인/공유리워드 체크리스트 확인
- [ ] UT 시뮬레이션 완료 및 리포트 작성

---

## STEP 6. 빌드 & 업로드하기

> 앱을 패키징하고 콘솔에 올리는 단계

| 스킬 | 설명 |
|------|------|
| `/ait-build` | typecheck → lint → ait build 순서로 빌드, 번들 용량 검증 및 콘솔 업로드 안내 |

- [ ] 빌드 성공 (`ait build`)
- [ ] `.ait` 번들 파일 생성 확인
- [ ] 번들 용량 100MB 이하 확인
- [ ] 콘솔에 `.ait` 파일 업로드
- [ ] CORS Origin 허용 목록 등록
  - 실서비스: `https://<appName>.apps.tossmini.com`
  - QR 테스트: `https://<appName>.private-apps.tossmini.com`

---

## STEP 7. 테스트 & 출시하기

> 토스앱에서 직접 테스트하고, 검토 요청 후 출시

| 스킬 | 설명 |
|------|------|
| (스킬 없음) | 이 단계는 콘솔에서 직접 수행 |

**테스트**

- [ ] QR 코드로 토스앱에서 테스트 (⚠️ 1회 이상 필수, 안 하면 검토 요청 불가)
- [ ] 앱이 정상적으로 열리는지 확인
- [ ] 인터랙션 반응 2초 이내 확인
- [ ] 종료 후 재진입 시 데이터 유지 확인
- [ ] (결제 있으면) 결제 흐름 테스트
- [ ] (권한 사용 시) 권한 미허용해도 나머지 기능 동작하는지 확인

**검토 요청 & 출시**

- [ ] 검토 요청 제출 (영업일 기준 최대 3일 소요)
- [ ] 검수 승인 이메일 수신 확인
- [ ] 콘솔에서 **'출시하기'** 클릭 → 즉시 전체 사용자에게 반영

---

## 출시 후 운영

- **새 버전 배포**: 새 번들 업로드 → 검토 → 승인 → 출시 (동일 프로세스)
- **롤백**: 콘솔에서 이전 버전 선택 → '출시하기'
- **긴급 수정**: 채널톡으로 즉시 문의
- **사후 검수**: 출시 후에도 앱인토스가 검수를 진행하며, 정책 위반 시 운영 중단될 수 있음
- **모니터링**: 오류/크래시 로그, API 응답 지연, 사용자 피드백 확인

---

## 한눈에 보기: 단계별 스킬 매핑

```
STEP 1  구상        →  /ait-plan, /ait-preflight
STEP 2  브랜딩      →  /ait-plan (이어서)
STEP 3  개발        →  /ait-scaffold → /ait-add-routing → /ait-add-query → /ait-tds-setup → /ait-implement
STEP 4  에셋        →  /ait-assets, /ait-meta
STEP 5  검수        →  /ait-review, /ait-ut
STEP 6  빌드        →  /ait-build
STEP 7  테스트·출시  →  (콘솔에서 직접)
```

전체 플로우를 한번에 실행하려면: `/ait-launch`
