# 2단계: 리소스 준비 (Resource Checklist)

## 목적

앱인토스 콘솔 등록 시 필요한 모든 이미지/텍스트 리소스를 사전에 준비한다.

## 필수 리소스 목록

### 이미지 리소스

| 항목            | 규격        | 형식                         | 필수            | 비고                                                |
| --------------- | ----------- | ---------------------------- | --------------- | --------------------------------------------------- |
| 앱 로고         | 600x600px   | PNG (배경색 필수, 투명 불가) | O               | 정사각형, 모서리 둥글기 불가, 다크/라이트 모두 대응 |
| 정방형 썸네일   | 1000x1000px | PNG                          | O               | 핵심 플레이/기능 화면                               |
| 가로형 썸네일   | 1932x828px  | PNG                          | O               | 핵심 플레이/기능 화면                               |
| 스크린샷 (세로) | 636x1048px  | PNG                          | 선택 (최소 3장) | 제출 시 우선 노출                                   |
| 스크린샷 (가로) | 1504x741px  | PNG                          | 선택 (최소 1장) | 제출 시 우선 노출                                   |

### 앱 로고 가이드라인

- 작은 화면에서도 **선명하게** 보여야 함
- 서비스 특성이 **직관적으로** 드러나야 함
- 다른 앱과 혼동되거나 **저작권 문제**가 없어야 함
- 다크/라이트 모드 모두에서 자연스러운 색상/형태
- 토스 제공 아이콘/이미지 리소스를 로고로 사용 불가 (2차 가공 포함)
- `granite.config.ts`의 `brand.icon`에 콘솔 업로드 후 URL 입력

### 썸네일 주의사항

- 핵심 플레이 화면이나 주요 기능이 보이는 이미지 사용
- 선명한 고화질, 작은 이미지를 늘려 쓰지 않기
- 정해진 규격에 맞춰 제작, 빈 공간을 색/블러로 채우지 않기
- 이미지는 중앙에 안정적으로 배치
- 비속어/정치적 표현 금지
- 과도한 텍스트 지양, 필요시 로고나 짧은 문구만
- 토스 제공 리소스를 썸네일로 사용 불가

### 텍스트 리소스

> **모든 텍스트 입력 필드 공통**: 특수문자는 `:`, `∙`, `?` 만 사용 가능. 이모지 사용 불가.

| 항목                    | 규칙                                                                               | 필수 |
| ----------------------- | ---------------------------------------------------------------------------------- | ---- |
| 한국어 앱 이름          | 서비스 성격이 바로 이해되는 이름, 특별한 이유 없으면 한글 표기 권장                | O    |
| 영어 앱 이름            | 15자 이내, 발음하기 쉬운 명사형, 명령문/동사구 불가                                | O    |
| appName (고유 ID)       | `intoss://` 스킴에 사용, **등록 후 수정 불가**, 영문 소문자+하이픈 권장            | O    |
| 카테고리                | 대분류 > 중분류 > 소분류 (비게임), 서비스와 맞지 않으면 검토 시 수정됨             | O    |
| 부제                    | 사용자가 얻는 가치를 구체적으로 표현, 비속어/느낌표 제외                           | O    |
| 상세 설명               | 사용자 경험 흐름(접속→행동→결과)을 구체적으로 기술, 선택이지만 홈 광고 선정에 활용 | 선택 |
| 앱 검색 키워드          | 사용자가 떠올릴 만한 키워드, 관련 없는 키워드 불가                                 | O    |
| 고객센터 이메일         | 원활한 문의 대응용                                                                 | O    |
| 고객센터 연락처         | 전화번호                                                                           | O    |
| 고객센터 채팅 상담 주소 | 채팅 상담 URL (채널톡 등)                                                          | O    |

### 영문 앱 이름 규칙 상세

- 15자 이내의 발음하기 쉬운 **명사형**
- 명령문이나 동사구 불가 ('Book a Taxi' X → 'GoRide Taxi' O)
- 한글 앱 이름과 의미가 다른 일반 단어만 단독 사용 불가 ('Taxi' X → 'GoRide Taxi' O)
- 단어 3개 이상이나 읽기 어려운 복합어 금지

### 상세 설명 작성 가이드

AI가 상세 설명을 바탕으로 토스 홈 광고 같은 마케팅 소재를 자동으로 생성함.
구체적으로 적을수록 더 다양하고 매력적인 광고가 만들어짐.

**작성 원칙:**

- 누구나 이해할 수 있는 쉬운 단어 사용
- 서비스 경험을 최대한 구체적으로, 많이 기술
- 사용자가 보고, 누르고, 경험하는 것을 흐름에 따라 설명

**예시 (맛집 탐색 서비스):**

> 거주 지역을 입력하면 근처 맛집 지도를 보여줘요. 맛집 지도에서 양식, 중식, 한식 등 음식 카테고리별로 검색할 수 있어요. 검색한 맛집에 하트 버튼을 누르면 찜할 수 있어요. 맘에 드는 맛집을 발견하면 버튼을 눌러 웨이팅을 바로 등록할 수 있어요.

### 브랜드 컬러

- 기존 브랜드 컬러가 있으면 그대로 사용
- 없으면 로고에서 가장 많이 사용된 색상 선택
- 6자리 HEX 코드 형식 (예: `#3182F6`)
- `granite.config.ts`의 `brand.primaryColor`에 입력

## 게임 전용 추가 리소스

| 항목                      | 비고                                                   |
| ------------------------- | ------------------------------------------------------ |
| 게임물 등급분류 증빙      | 게관위 심의 증명서 PDF 또는 스토어 URL                 |
| 자체등급분류 게임물 정보  | 등록자명, 사업자명, 등급분류일자, 번호, 등급, 내용정보 |
| 대표자 인감/사인 이미지   | 흰 종이에 인감 또는 서명                               |
| 게임 플레이 화면 스크린샷 | 기존 스토어용 2장 + 앱인토스용 2장                     |
| 리더보드 설정             | 점수 단위, 정렬 기준 등                                |

## 비게임 전용 추가 요구사항

- **앱 내 기능**: 최소 1개 이상 등록 필수
  - 예: '식단 기록하기' → 홈 화면을 건너뛰고 바로 해당 기능 화면으로 이동
