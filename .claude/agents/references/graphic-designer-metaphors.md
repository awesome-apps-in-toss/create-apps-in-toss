로고 후보를 떠올릴 때 참고하는 은유 테이블과 2축 흡수 주의사항, 식별성 테스트 예시 모음.
에이전트는 로고 워크플로 3단계(후보 떠올리기) 직전에 Read 로 열어 참고한다.

## 2축 앱 주의

앱이 "카테고리 A × 행동/유형 B" 의 교차점이면 두 축 모두 시각적으로 표현해야 합니다. 한 축만 강조하면 인접 카테고리로 **흡수**됩니다:

| 앱 정체성 | 단일 축 강조 시 흡수될 카테고리 |
|----------|--------------------------------|
| 맛집 × 검색 | "맛집"만 → 요리. "검색"만 → 일반 검색 |
| 운동 × 코칭 | "운동"만 → 피트니스 장비. "코칭"만 → 자기계발 |
| 장난감 × 교육 | "장난감"만 → 쇼핑몰. "교육"만 → 학습지 |
| 유치원 × 선택 테스트 | "유치원"만 → 부동산/기관. "테스트"만 → 일반 퀴즈 |
| 반려동물 × 펀딩 | "반려"만 → 동물 용품. "펀딩"만 → 기부 일반 |
| 여행 × 일정 | "여행"만 → 관광지. "일정"만 → 캘린더 |
| 음악 × 운동 | "음악"만 → 뮤직플레이어. "운동"만 → 피트니스 |

**해법**: 각 축을 구현하는 시각 기호를 분리해 하나의 캐릭터에 결합하거나, 캐릭터 + 소품 구조로 병치.
- 예: 맛집 × 검색 → 접시(대상) + 돋보기(행동)
- 예: 운동 × 코칭 → 러닝 실루엣(대상) + 스톱워치(행동)
- 예: 반려동물 × 펀딩 → 발바닥(대상) + 감싸는 손/하트(행동)

단, 1개 기호가 너무 많은 축을 떠받으면 "잡화" 로 보이니 **축당 1~2개 기호**만 담고 나머지는 과감히 제거.

## 식별성 테스트 예시

워크플로 7-c 단계(카테고리 식별성 테스트) 에서 "5개 후보 나열 + 상위 2개에 실제 카테고리 포함" 판정 방식을 이해하기 위한 예시 모음.

- 예 1 (유치원 선택 테스트 앱):
  * "집 + 분기 화살표" → 부동산·이사·네비게이션·주거선택·유치원 (유치원 5등) → **탈락**
  * "얼굴 없는 흰 인체 + ?" → Q&A·FAQ·랜덤박스·미아찾기·심리테스트 → placeholder 연상 다수 → **탈락**
  * "유아 실루엣(1:1비례) + 웃는 얼굴 + 체크/카테고리 요소" → 교육·육아·성향테스트·어린이서비스·유치원 → 상위 2~3등 → **통과**
- 예 2 (운동 습관 기록 앱):
  * "체크마크만" → 할일·투표·설문·독서기록·운동 → 상위에 할일 우세 → **탈락**
  * "러닝 실루엣 + 캘린더 + 체크" → 피트니스·운동·습관·루틴·건강 → 상위 2등 모두 해당 → **통과**
- 예 3 (맛집 추천 앱):
  * "지도 핀만" → 지도·여행·배달·부동산·맛집 → 지도·여행 우세 → **탈락**
  * "접시 + 별점 + 핀" → 맛집·리뷰·식당예약 → 상위 모두 해당 → **통과**
