# Dashboard Run Status Contract

대시보드에서 실행되는 skill은 종료 직전 구조화된 상태 파일을 반드시 기록한다.
이 파일 경로는 환경변수 `AIT_RUN_STATUS_PATH` 로 전달된다.

## 규약

- 사용자에게 `✅` / `❌` 텍스트를 출력하기 전에 먼저 상태 파일을 기록한다.
- 환경변수가 없으면(대시보드 밖 실행) 상태 파일 기록은 생략 가능하다.
- 성공 시 JSON:

```json
{"status":"success"}
```

- 실패 시 JSON:

```json
{"status":"failure","reason":"<한 줄 원인>"}
```

## 필수 원칙

1. 성공/실패는 한 번만 결정한다.
2. 마지막에 질문 대기 상태로 끝내지 않는다.
3. 완료 가능한 상태면 추가 확인 질문 없이 저장/반영 후 종료한다.
4. failure reason 은 대시보드 사용자가 바로 이해할 수 있는 한 줄이어야 한다.
5. Write 실패나 JSON 손상도 실패로 간주한다.

## 권장 종료 순서

1. 산출물 저장 또는 필요한 반영 수행
2. `AIT_RUN_STATUS_PATH` 에 상태 JSON 기록
3. 짧은 완료/실패 보고 1회 출력
4. 종료
